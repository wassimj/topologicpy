Formal ontology specification

TopologicPy Ontology

A semantic framework for representing geometry, topology, graphs, building spaces, elements, provenance, and analysis metrics in TopologicPy workflows.

Download current TTL Version 0.1.0 GitHub repository

Canonical namespace

The ontology uses a persistent w3id namespace. The physical ontology file is hosted through GitHub Pages, but the namespace is intended to remain stable. 

@prefix top: <http://w3id.org/topologicpy#> .

Example usage

Use the namespace in Turtle, RDF, graph databases, linked-data exports, and semantic annotations. 

@prefix top: <http://w3id.org/topologicpy#> .

:room_101 a top:Room ;
    top:hasArea "24.6"^^xsd:double ;
    top:adjacentTo :corridor_1 .

:wall_12 a top:Wall ;
    top:bounds :room_101 .

Implementation in TopologicPy

The Python implementation uses dictionaries as the lightweight semantic layer. 

from topologicpy.Ontology import Ontology

Ontology.SetClass(cell, "top:Room")
Ontology.SetLabel(cell, "Room 101")
Ontology.SetCategory(cell, "space")

ttl = Ontology.TTLString(cell)

Topology

Defines core TopologicPy entities and compositional relations such as containment, boundaries, subtopologies, interfaces, and apertures.

Buildings

Represents spatial and architectural concepts including sites, buildings, storeys, spaces, rooms, thermal zones, and building elements.

Graphs

Supports graph-native workflows including nodes, relationships, adjacency graphs, visibility graphs, knowledge graphs, and graph analysis metrics.

Alignment

The ontology is intended to interoperate with linked building data and semantic web ecosystems.

  • BOT
  • Brick Schema
  • RDF
  • RDFS
  • OWL
  • IFC
  • Linked Building Data

Persistent identifier policy

The preferred namespace is http://w3id.org/topologicpy#. The redirect points to the GitHub Pages-hosted ontology document. If hosting changes, the redirect can be updated without changing published data.
